运算符重载

由于重载函数是Complex类中的对象，有一个参数是隐含的，运算符函数用this指针隐式访问类对象的成员。在上面，重载函数operator+访问了两个对象中的成员，一个是this指针指向的对象中的成员，一个是形参对象中的成员。如this->real + c2, this->real就是c1.real。

  1. 不允许定义新的运算符，只能对已有的运算符重载
  2. 不能重载的运算符有五个：
  • .
  • . *
  • ::
  • sizeof
  • ?:
  1. 重载不能改变运算符运算对象的个数
  2. 重载不能改变优先级
  3. 重载不能改变结合性
  4. 重载运算符的函数不能有默认的参数
  5. 重载的运算符必须和用户定义的自定义类型的对象一起使用

重载流插入运算符和流提取运算符

对”<<”和”>>”重载的函数形式如下
istream & operator >> (istream &, 自定义类 &)；
ostream & operator >> (ostream &, 自定义类 &)；
第一个参数和函数必须是istream&或ostream&类型，第二个是要输出的类。所以重载的函数只能作为友元或者普通函数，不可以作为成员函数。

用”<<”输出用户声明类的对象的信息，以上面友元重载的复数操作作为例子

详看注释，return output语句就是return cout，执行流程为：
cout << c3;
cout << c3的返回值是cout的当前值，之后
(cout << c3) << c2; // 该式子左侧也是ostram类对象，右侧是Complex类对象
所以解释了为什么C++规定”<<”的第一个参数和函数的类型必须是ostream，在返回cout对象之后可以连续输出。
eg. cout << c3 << 5 <<endl; 前面部分才是重载的流插入符，因为右侧不是Complex类的对象。
